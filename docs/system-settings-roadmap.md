# 系统设置扩展建议

## 目标

系统设置不只负责“换颜色”，还应该同时覆盖视觉风格、布局层级、交互节奏、性能策略和辅助能力，形成一套真正可运营的前端外观系统。

## 建议优先新增

### 1. 内容容器层级

- 提供 `轻盈 / 标准 / 强卡片` 三档
- 控制主体容器的边框、阴影、背景透明度、内外边距
- 首页、工作台、表格页可按场景覆盖

价值：

- 直接解决“页面像很多层叠放”的问题
- 让 `default`、`future-tech`、节日主题都能有不同的空间层次

### 2. 视觉密度

- 提供 `紧凑 / 舒适 / 宽松` 三档
- 统一影响表单高度、表格行高、工具栏间距、卡片 padding、弹层间距

价值：

- 大屏运维场景可以更紧凑
- 展示型首页和节日主题可以更舒展

### 3. 动效强度

- 提供 `关闭 / 轻量 / 标准 / 强化` 四档
- 控制菜单过渡、卡片 hover、主题切换、边框呼吸、粒子装饰

价值：

- 低性能机器可以直接切到轻量
- 未来科技主题可以保留氛围但不拖慢主线程

### 4. 性能优先模式

- 提供 `视觉优先 / 平衡 / 性能优先` 三档
- 自动联动以下开关：
  - 背景模糊
  - 发光阴影
  - 动态粒子/装饰
  - FPS 监控刷新频率
  - 首页动画与图层数量

价值：

- 把“卡不卡”从隐性问题变成显性设置
- 方便低端环境和远程桌面场景快速切换

### 5. 首页模式

- 提供 `仪表板 / 轻工作台 / 极简欢迎页`
- 支持控制：首页头部信息、统计卡、空态样式、快捷入口区块

价值：

- 首页不再只有一种样子
- 能兼顾“操作型用户”和“展示型用户”

### 6. 导航细节

- 抽屉导航二级菜单宽度档位
- 悬停导航触发延迟
- 折叠菜单图标尺寸
- 菜单文本单行省略 / 自动换行开关

价值：

- 解决长标题、二级菜单、不同主题下导航风格不一致的问题

## 第二阶段建议

### 7. 顶部工具栏编排

- 支持拖拽排序
- 支持显示/隐藏具体工具项
- 支持时钟样式、语言下拉样式、头像区密度切换

### 8. 主题细化能力

- 每个主题单独配置：
  - 主色
  - 辅助色
  - 高亮色
  - 容器圆角
  - 图标风格
  - 装饰强度

价值：

- 主题不再只是换皮，而是完整的视觉系统

### 9. 页面级个性化

- 支持页面级覆盖：
  - 是否启用卡片外壳
  - 是否展示背景纹理
  - 是否启用工具提示风格增强

价值：

- 首页、系统设置、个人信息页可以更精细地调性分层

### 10. 无障碍与辅助

- 高对比模式
- 大字体模式
- 减少动画
- 读屏优化
- 色弱友好主题

## 推荐落地顺序

1. 内容容器层级
2. 视觉密度
3. 动效强度
4. 性能优先模式
5. 首页模式
6. 导航细节

## 本轮最值得先做的 3 个开关

1. `内容容器层级`
2. `性能优先模式`
3. `首页模式`

这 3 个开关能同时解决当前最明显的三个痛点：层级太重、未来科技易卡、首页缺少场景化配置。
