# 数据同步系统整合 Job 调度模块 - 需求文档

## 功能概述

将 `spring-support-sync-data-starter` 的定时调度功能整合到 `spring-support-job-starter` 统一调度系统中，实现统一的任务管理、监控、日志记录和告警机制。

## 业务需求

### 1. 统一调度管理

- 同步任务的定时调度统一使用 Job 系统
- 保持现有的 Cron 表达式和间隔调度配置
- 支持手动执行和定时执行两种模式

### 2. 统一日志记录

- 执行日志统一记录到 Job 日志系统
- 保留同步任务特有的统计信息（读取/写入/成功/失败数量）
- 支持实时日志详情查看

### 3. 统一监控告警

- 任务执行状态统一监控
- 失败重试机制统一管理
- 支持邮件告警（通过 Job 系统）

### 4. 向后兼容

- 通过配置开关控制是否使用 Job 系统
- 不影响现有的同步任务功能
- 平滑迁移现有任务

## 功能范围

### 包含功能

1. 创建 SyncTaskJobHandler 包装同步任务执行逻辑
2. 修改任务启动/停止逻辑，集成 Job 系统
3. 统一日志记录到 Job 日志表
4. 配置开关控制集成方式
5. 任务状态同步（Job ↔ SyncTask）

### 不包含功能

1. 前端界面修改（本期仅后端整合）
2. 现有同步任务的数据迁移
3. Job 系统的功能增强

## 验收标准

1. 启用 Job 集成后，同步任务通过 Job 系统调度执行
2. 任务执行日志同时记录到 `sys_job_log` 和 `monitor_sync_task_log`
3. 任务启动/停止操作同步更新 Job 状态
4. 禁用 Job 集成后，系统回退到原有调度方式
5. 所有现有功能正常工作，无回归问题

## 技术约束

1. 保持现有 API 接口不变
2. 数据库表结构不变
3. 不影响现有的 WebSocket 实时推送
4. 支持 Spring Boot 3.x

## 风险评估

1. **兼容性风险**：Job 系统和 Sync 系统的调度机制差异
   - 缓解措施：通过适配器模式封装差异
2. **性能风险**：双写日志可能影响性能
   - 缓解措施：异步写入，批量提交

3. **数据一致性风险**：两个系统的状态可能不同步
   - 缓解措施：事务控制，状态同步机制
