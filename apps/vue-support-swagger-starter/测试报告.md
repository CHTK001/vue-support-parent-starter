## 部署流程

### 1. 构建 Vue 项目并部署

在 `vue-support-swagger-starter` 目录下运行：

```bash
cd h:\workspace\2\vue-support-parent-starter\apps\vue-support-swagger-starter
pnpm deploy
```

### 2. 构建 Spring 项目

```bash
cd h:\workspace\2\spring-support-parent-starter
mvn clean install -DskipTests -pl spring-support-swagger-starter -am
```

### 3. 启动 System 服务

```bash
cd h:\workspace\2\spring-support-api-parent\spring-api-support-system-starter
java -jar target/spring-api-support-system-starter-4.0.0.37.jar
```

## 注意事项

1. **认证问题**：部分路径（如 `/doc.html`、`/doc-v2`）可能需要登录认证，这取决于 Spring Security 配置。

2. **Context Path**：不同服务的 context path 不同：
   - System 服务：`/system/api`
   - OAuth 服务：`/oauth`（Armeria，非标准 Web）

3. **端口配置**：
   - System 服务：18170
   - OAuth 服务：19180（Armeria）

4. **部署顺序**：
   - 先运行 `pnpm deploy` 部署 Vue 文件
   - 再构建 Spring 项目
   - 最后启动服务
